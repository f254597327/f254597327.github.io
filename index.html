2016最新翻墙姿势

发表于 2016-01-05   |  

前言：

现在的翻墙软件也学会做全家桶了。简而言之就是全平台都要有客户端，一般情况下，只要有一个平台的能用，就全平台能通用。我不推荐使用自由门和无界。原因就是它们看起来与政治走的很近。也不推荐使用不了解背景的vpn，原因是安全。此时此刻在我所在的地区，openvpn，ssh，fqrouter，vpngate，无界，自由门皆以失效，你们最少备上两个，以备不时之需。

桌面端：

  1. Lantern。第一当之无愧。2.0版本刚出。youtube720P的视频也是小菜一碟。缺点就是内部有黑名单，很多色网和bt资源的网站打不开。
  2. psiphon3。大行齐道好多年。依然不减当年。
  3. xxnet+openshift。不稳定。速度快。

安卓端：

  1. paiphon3，速度快，支持全局代理。apk直接安装运行即可，省心。

  2. lantern.apk,速度第一，但是需要自己编译。网上的apk总觉得来路不正，有安全隐患。

  3. FreeBrowser。速度还行，仅是个浏览器，不是全局代理。

IOS端

  1. betternet，速度第一,稳定可靠。缺点需要一个美国的icloud账号，去美国的appstore下载。安装的时候需要提前翻墙，以后就不需要了。

  2. Onavo Protect，大致与上面类似。不稳定。时常掉线。安全性不错。

通用版

直接修改hosts文件，但这种方式桌面端还可以，安卓和苹果都需要有最高权限，我是感觉弊大于利。不推荐。但直连无疑是速度最快的。只要GFW不封锁该IP，就能一直用，缺点也在于此。所以需要备有稀有ip，用到的时候，直接更换。

为什么修改了hosts文件后可以顺利访问youtube但不能看上面的视频

发表于 2016-01-04   |  

包括Youtube页面在内的大部分Google服务都可以通过任何一个gws来完成,也即我们可以找到一个gws的IP地址,然后把Google的所有域名都泛解析上去即可满足大部分的访问需求,但Youtube视频是一个例外.Youtube内的所有视频全部是通过http://googlevideo.com这一域名完成的,并且IP地址和域名是一一对应的.而且某个视频只存在于某些服务器上,在用户请求的环境下不能够通过Google的内网传输.这就意味着单纯地把http://googlevideo.com泛解析到某一个Google的IP地址是无法观看视频的.
Youtube视频在请求时,首先会将你direct到某一组gvs上,我们在解决了DNS污染问题的前提下通过访问https://redirector.googlevideo.com/report_mapping 即可得知我们被direct到了哪组服务器上,例如我这里是hkg08s08.通过解析r1.hkg08s08.googlevideo.com即可得到服务器的IP地址.然后把这一组域名及对应的IP地址一一写入hosts中,在IP地址没有被封锁的情况下即可访问视频.但由于direct服务器的变化及在不能访问时Youtube会做的二次direct,这样做并不能保证可以稳定访问所有视频,因为并没有完全解除对googlevideo.com的DNS污染.
想要通过解决DNS污染的方式来观看Youtube,最可行的办法是通过bind或dnsmasq来架设本地DNS服务器,并把http://googlevideo.com这一个域名forward到一个能够返回正确IP地址的服务器(例如opendns的5353端口).由于googlevideo从2014年初开始支持了https方式访问,所以通过这种手段可以做到Youtube的直连.但这种方法在IP地址被严厉封锁的时期依然不能观看视频,最近一年就是时好时坏.即使可以观看由于高峰时期拥堵的国际线路,视频的加载速度在大部分地区也不会十分理想.在现阶段最好的解决办法依然是寻找购买优质的VPS,但这就是另外的问题了.

PS:gws:Google Web Server; gvs:Google Video Server.

PS:在观看一个视频时，大致步骤如下：

浏览器首先向 youtube.com 发起请求，youtube.com 会根据来源 IP 所属的 IP 段向客户端分配一个服务器 (通常会选择性能最好的节点，但是因为某些原因，会出现 Issue #8 描述的现象)，比如 r1—sn-gxuxapu-hm2e.googlevideo.com。
客户端向r1.sn-gxuxapu-hm2e 请求真正的视频文件。
如果此服务器无法访问，youtube.com 进行第二次 redirect，如 r8—sn-o097znel.googlevideo.com。
客户端向第二次分配的服务器 r8.sn-o097znel 请求视频文件。
如果第二次分配的服务器还无法访问，那么失败。
第一步是根据来源 IP 进行导向的，并且 sn 域名全是固定 IP，所以无法进行重定向来加速。除非用代理访问 youtube.com 这个域名 (间接改变来源 IP)，然后直连分配的 googlevideo 服务器。

依上面的思路。捣鼓了一天，直接观看youtube宣告失败。原因在于封锁IP。

寻找谷歌ip

发表于 2015-12-30   |  

寻找稀有谷歌ip

逐个查找与谷歌网域关联的DNS记录

nslookup -q=TXT _netblocks.google.com 8.8.8.8
nslookup -q=TXT _netblocks2.google.com 8.8.8.8
nslookup -q=TXT _netblocks3.google.com 8.8.8.8

结果如下：
“v=spf1 ip4:64.18.0.0/20 ip4:64.233.160.0/19 ip4:66.102.0.0/20 ip4:66.249.80.0/20 ip4:72.14.192.0/18 ip4:74.125.0.0/16 ip4:108.177.8.0/21 ip4:173.194.0.0/16 ip4:207.126.144.0/20 ip4:209.85.128.0/17 ip4:216.58.192.0/19 ip4:216.239.32.0/19 ~all”

这并不是 Google 保留的 IP 地址的完整列表。。

nmap扫描审查端口443

nmap -sS -sV -p 443 -Pn 谷歌ip段

此步骤就要拼人气了。喝口茶，回来再看看。结果是这样的就能用

443/tcp open ssl/http Google httpd 2.0 (GFE)

markdown语法编辑器

发表于 2015-12-23   |  

搭建完gihub+hexo博客之后，开始写博客，发现在WINDOWS上没有趁手的编辑器，网上推荐的，一一试过，问题多多。比如有的不能在线预览，有的不能导出，有的出现乱码。最后选择了atom。

为什么选择atom？

  1. 免费。俺只写博客，不搞软件工程，用不着重量级的编辑器。
  2. 易用。相比vim，配置起来简单，不用和指令打交道，没有鸡肋功能。有一些编辑器能直接发布到网上且不能本地保存，麻烦多多。
  3. activate-power-mode这个插件太炫了。写博客肯定很爽。

为atom安装markdown插件。

  1. markdown-preview，此插件内置。 shirft+ctrl+m 呼出预览。

  2. markdown-writer，需要去packeges里安装。自动激活.

    “shift-ctrl-K”: “markdown-writer:insert-link

    “shift-ctrl-I”: “markdown-writer:insert-image”

    “ctrl-i”: “markdown-writer:toggle-italic-text”

    “ctrl-b”: “markdown-writer:toggle-bold-text”

    “ctrl-‘“: “markdown-writer:toggle-code-text”

    ‘ctrl-‘’: “markdown-writer:toggle-codeblock-text”

    “ctrl-h”: “markdown-writer:toggle-strikethrough-text”

    “ctrl-1”: “markdown-writer:toggle-h1”

    “ctrl-2”: “markdown-writer:toggle-h2”

    “ctrl-3”: “markdown-writer:toggle-h3”

    “ctrl-4”: “markdown-writer:toggle-h4”

    “ctrl-5”: “markdown-writer:toggle-h5”

开始写作

  • markdown语法
  • markdown-writer使用方法
